# For a Special Ghoul 🌹

**For a Special Ghoul** is an interactive, multi-page web experience inspired by the manga and anime series *Tokyo Ghoul* by Sui Ishida. Created as a creative "Valentine's proposal," the project guides users through a narrative journey—from receiving a sealed envelope to a final interactive question.

## 🚀 Professional Overview
I developed this project to demonstrate foundational proficiency in **HTML5** and **CSS3**, specifically focusing on user flow, thematic UI consistency, and responsive element positioning. It serves as a practical application of front-end basics, showing how code can be used to create a branded, storytelling experience.

## 🛠️ Technical Implementation

### **Layout & Design**
* **Centering & Positioning:** Utilized `absolute` positioning combined with `transform: translate(-50%, -50%)` to ensure content remains perfectly centered across various screen sizes.
* **Flexbox Navigation:** Implemented `display: flex` and `justify-content` in the interaction footer to manage the alignment of action buttons.
* **Thematic UI:** Applied a global aesthetic using **Ink Free** typography and a warm `#efdecd` background to simulate a personal, handwritten letter.
* **Visual Polish:** Enhanced imagery with `border-radius: 10px` and `box-shadow` to create a modern, "elevated" feel for the assets.

### **Interactive Flow**
The site uses a logic-driven path to move the user through the experience:
1.  **Entry (`index.html`):** A call-to-action (CTA) to open a sealed envelope.
2.  **Narrative (`letter.html`):** A content-heavy page focusing on readability and layout.
3.  **The Question (`question.html`):** The core interaction using Vanilla JavaScript `onclick` events for navigation.
4.  **The "Logic Trap" (`no.html`):** A playful "404" page that prevents a "No" response, demonstrating creative UX redirection.

## 📂 Project Architecture
* **HTML5:** Structure and interconnected page routing across five distinct pages.
* **CSS3:** Centralized styling via `styles.css` for site-wide visual identity.
* **Assets:** Curated images that align with the *Tokyo Ghoul* theme.

## 📈 Growth & Future Iterations
To show my progression as a developer, I am targeting the following improvements for future versions:
* **Refactoring:** Transitioning from absolute positioning to **CSS Grid** for more robust mobile responsiveness.
* **Clean Code:** Resolving minor syntax artifacts in the stylesheet (such as the redundant closing brace) to maintain 100% CSS validation.
* **Accessibility:** Adding `aria-labels` to interactive buttons to improve screen-reader compatibility.
